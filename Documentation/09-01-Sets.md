# Sets

What if you have a list with duplicates and you want to automatically remove them? What if you want to find items that two lists have in common? Sets handle both of these. 🎯

A set is a collection with **no duplicates** and **no guaranteed order**.

---

# Set Syntax

Sets use curly braces `{}` — same as dictionaries, but with no key-value pairs:

```py
genres = {'rock', 'pop', 'jazz', 'rock', 'pop'}
print(genres)

# Output: {'rock', 'pop', 'jazz'}  (duplicates removed, order may vary)
```

Or create a set from a list:

```py
playlist = ['ABBA', 'Queen', 'ABBA', 'Bowie', 'Queen', 'Queen']
unique = set(playlist)
print(unique)

# Output: {'ABBA', 'Queen', 'Bowie'}  (no duplicates)
```

---

# Key Properties

- **No duplicates** — adding an existing element does nothing
- **Unordered** — you can't access items by index (`my_set[0]` crashes)
- **Mutable** — you can add and remove items
- **Fast membership testing** — checking `if x in set` is much faster than `if x in list` for large collections

---

# Adding and Removing

```py
languages = {'Python', 'JavaScript', 'Go'}

languages.add('Rust')
print(languages)    # Output: {'Python', 'JavaScript', 'Go', 'Rust'}

languages.remove('Go')       # raises KeyError if not found
languages.discard('C++')     # safe — does nothing if not found
```

Use `.discard()` when you're not sure if the item exists.

---

# Set Operations

This is where sets get really powerful. Think back to math class — Venn diagrams. 📊

**Union** `|` — everything from both sets:

```py
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a | b)    # Output: {1, 2, 3, 4, 5, 6}
```

**Intersection** `&` — only what's in both:

```py
print(a & b)    # Output: {3, 4}
```

**Difference** `-` — what's in `a` but not in `b`:

```py
print(a - b)    # Output: {1, 2}
```

**Symmetric Difference** `^` — what's in one but not both:

```py
print(a ^ b)    # Output: {1, 2, 5, 6}
```

playlist_a = ['MGMT - Little Dark Age', 'Joji - YUKON', 'Rihanna - Only Girl',
              'Dean Martin - Everybody Loves Somebody', 'MGMT - Little Dark Age',

# List vs Tuple vs Set — When to Use Which

| | List | Tuple | Set |
|---|---|---|---|
| Ordered? | ✅ | ✅ | ❌ |
| Duplicates? | ✅ | ✅ | ❌ |
| Mutable? | ✅ | ❌ | ✅ |
| Index access? | ✅ | ✅ | ❌ |
| Use when | Ordered, changeable data | Fixed, unchanging data | Unique items, set math |

> [!TIP]
> If you just want to deduplicate a list, the one-liner is `list(set(my_list))`. Convert to set (removes duplicates), then back to list. Note: this loses the original order.

# Chapter 9 Recap

- Sets: `{item1, item2}` — no duplicates, no order.
- Create from a list: `set(my_list)`
- `.add()` to add, `.remove()` or `.discard()` to remove.
- Set math: `|` union, `&` intersection, `-` difference, `^` symmetric difference.
- Fast membership testing with `in`.
- Can't access by index — no `set[0]`.
